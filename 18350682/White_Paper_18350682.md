# Universal Coherence Scoring Protocol (UCSP)
## A domain-agnostic method to evaluate coherence without inventing numbers

### DOI
10.5281/zenodo.18350682

### Abstract
This white paper specifies a universal, falsifiable protocol to evaluate a level of coherence for any opered (object, information, system, model, experiment). The method explicitly separates the law (a non-anthropocentric admissibility constraint) from its contextual applications, and forbids validation-by-plausibility. No numerical value may enter the equation unless it is measured, sourced, or explicitly left symbolic, in which case the only valid output is indeterminate. Coherence is evaluated across four non-compensatory planes (P1–P4) and aggregated by a strict minimum, preventing “good-looking” dimensions from masking structural failure modes. A fully specified worked pattern (hammer) illustrates that scientific rigor includes explicit indeterminacy when critical measurements are missing. Three Zenodo-ready micro-examples (information, generative AI, transversal systemic evaluation) show how the same protocol can be applied across domains while remaining strictly within the mathematical frame.

---

## 1. Problem
Many evaluation practices confuse surface regularity, local performance, and global admissibility. In written arguments and AI-generated content, fluency often triggers confirmation: if a statement “sounds coherent”, it is treated as validated. This creates drift: narratives become self-reinforcing, and missing data is silently replaced by plausible numbers or implicit assumptions.

This document addresses that failure mode by enforcing a strict, reproducible, and falsifiable protocol: a coherence evaluation is valid only if every variable is defined, measured or sourced, or explicitly declared missing.

## 2. Theoretical status of the law
### 2.1 Constraint, not explanation
The coherence law in this document is an admissibility constraint, not an explanatory or predictive model. It does not claim why a system persists; it formalizes conditions without which persistence cannot be observed within a given frame.

### 2.2 Continuous quantity, not a binary label
This framework does not use “coherent/incoherent” as binary categories. Coherence is treated as a continuous quantity compared to thresholds in an explicit frame. A system is never “incoherent in itself”; it can only have coherence insufficient for a specified frame, use, and duration.

### 2.3 Foundational circularity (non-vicious)
The apparent circularity of coherence filtering coherent states is not a logical weakness but a foundational circularity typical of conditions of possibility. The law is not used to justify itself; it only bounds admissible states.

### 2.4 Non-anthropocentric law; anthropocentric applications
The law is non-anthropocentric and applies across physical, informational, biological, and social systems without introducing intention or moral value. When humans intervene as operators or as the population concerned, an anthropocentric index may be added as a contextual annotation, without altering the law.

## 3. Operational decomposition: planes P1 to P4
A coherence evaluation must specify the plane(s) being used; otherwise, the result has no operational meaning.

### P1 — Intrinsic (internal admissibility)
P1 evaluates internal integrity and the absence of structural contradictions, independent of use, population, or context.

### P2 — Contextual (function in a declared frame)
P2 evaluates the ability to satisfy a declared function in an explicit usage context under explicit constraints.

### P3 — Population / market (compatibility threshold)
P3 evaluates the proportion of a defined population able to reach “valid use” without excessive compensation. When required, a threshold is imposed (e.g., 0.80).

### P4 — Systemic (global effects)
P4 evaluates global effects on adjacent systems, environment, or long-term stability. P4 is not a moral judgement; it is an admissibility constraint on externalities and systemic destabilization.

### Non-compensation
P1–P4 are analytically independent and may not compensate each other. A weakness in a single plane cannot be “offset” by strength elsewhere.

---

## 4. Mathematical engine and normalization
### 4.1 Strict aggregation
Let each plane yield a bounded score C_Pk in [0,1], or the value INDETERMINATE if critical data is missing.

**Canonical aggregation:**
\[
\boxed{ C(X\mid F) = \min\big(C_{P1},C_{P2},C_{P3},C_{P4}\big) }
\]

This aggregation prevents compensatory averaging and forces explicit identification of limiting planes.

### 4.2 Allowed outputs only
A coherence evaluation may output only:
- Computed (all required variables measured or sourced),
- Bounded (interval [a,b] due to uncertainty),
- Indeterminate (critical data missing; no invented values).

Any attempt to introduce a non-measured, non-sourced, or non-bounded value invalidates the computation and forces an indeterminate output.

### 4.3 Continuous regimes
Coherence is interpreted relative to thresholds in a frame. In particular, “coherence insufficient” is meaningful; “incoherent” is not.

## 5. Application methodology: Phase A / Phase B
### 5.1 Phase A (structure without numbers)
Phase A defines the opered, the explicit frame, the planes involved, and the full variable table with units. No numerical value is introduced.

Any variable without a measurement or sourcing method remains symbolic; if it is critical, the evaluation must be indeterminate.

### 5.2 Phase B (measurement or sourcing)
Phase B fills Phase A variables using measured data or explicit sources. If multiple sources exist, variability is preserved as an interval rather than collapsed into a compensatory mean.

### 5.3 Testable negation
Every application must include a testable negation stating under which conditions the coherence would become insufficient or indeterminate.

## 6. Worked pattern: the hammer (reference example)
The hammer is used as a reference pattern, not because it is intrinsically important, but because its function, constraints, and failure modes are easy to define.

### 6.1 Opered and frame (canonical)
- Opered: a mass-produced consumer hammer.
- Function: drive standard nails into a common material.
- Population: non-specialized users (P3 threshold).
- Duration: explicit but domain-dependent.
- Plan selection: P1 (integrity), P2 (functional use), P3 (population). P4 optional.

### 6.2 Why the global output can be indeterminate
If structural integrity variables (e.g., pull-out resistance of head/handle, fatigue cycles) are not measured or sourced, P1 is indeterminate. Because aggregation is a minimum, the global result must be indeterminate, even if P2 and P3 are measurable.

This is not a weakness; it is the intended scientific behavior: refusing to produce a score when critical evidence is missing.

---

## 7. Expected results and scope
If applied rigorously, the protocol yields:
- Explicit distinction between coherence insufficient, functional, stable, and excess regimes in a declared frame.
- Comparability across heterogeneous opereds when planes and variables are aligned.
- Explicit refusal of unjustified conclusions (indeterminate as valid output).
- Identification of limiting planes and variables without prescribing solutions.

The framework does not predict, optimize, or decide. It bounds admissibility under explicit constraints.

---

## 8. Discussion: limits, non-promises, extensions
### 8.1 Limits
The framework is limited by data availability, measurability, and explicit definition of frames. These limits are not patched by plausible values; they are expressed as indeterminacy.

### 8.2 Non-promises
The framework does not measure meaning, psychological intention, or moral value. It does not replace human or institutional judgement.

### 8.3 Extension: construction by inversion (method derived)
The same constraints used for evaluation can guide construction by controlled inversion: identify the limiting plane (the minimum), then adjust only variables attached to that plane until a target threshold is met, or conclude indeterminacy/impossibility in the frame. This is a constraints-based engineering procedure, not an optimization promise.
